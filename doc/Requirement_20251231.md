# 需求說明
我需要開發一個版本比對的小工具，目的是為了要讓我在進行程式上版時能快速的列出，`本次上版程式`與`上次上版程式`程式的差異清單，因此需要一個小程式，讓我可以輸入相同分支的兩個 Commit，按下執行後會去搜尋差異的檔案清單列表，相關詳細需求如下：


# 輸入欄位需求
1. Git 網址，例如 https://tfs.lingcheng.tw/tfs/DefaultCollection/PJ2025069/_git/Web
2. Azure Devops / TFS 的 PAT 個人權杖
3. 基準 Commit ID，例如 6d7bdd0e040be168463fa6f92d82c41a83465612，允許輸入短碼
4. 比對 Commit ID，例如 6d7bdd0e040be168463fa6f92d82c41a83465612，允許輸入短碼
5. 送出比對後，要將成功後的輸入欄位記錄下來，下次重新近來需記住最後輸入的資訊。
6. 輸入框要提供 placeholder 說明。


# 輸出呈現需求
1. 如果基準 Commit 與比對 Commit 之間還有多個必須列出至基礎開始到目前為止的所有檔案差異。
2. 資料呈現方式必須明確的可以看出是哪個資料夾下的檔案，並且須依造資料夾和檔案名稱排序，相同的資料夾必須放在一起。
3. 資料列表須能明確地看出檔案的狀態，例如新增、修改、刪除。
4. 提供將結果匯出的功能，需匯出 .csv 格式，所需欄位如上述。
5. 若比對正在處理中，請鎖定 UI 不能讓使用者修改，並且要有類似 progress 進度條或者顯示相關處理狀態的說明或進度視窗，避免使用者以為程式當機，並在處理完畢後恢復可輸入狀態。


# 技術需求
1. 請使用 WindowsForm 並使用 NET Core 8 或者 10 等 LTS 版本下去開發。
2. 若要使用套件，請務必使用允許商用版本的套件。
3. 請遵循 Clean Code 的原則，另外若有特定功能的處理類別，例如處理 Git 歷程等，也請獨立類別。
4. 各類別、屬性、方法等 DOC 請使用繁體中文進行說明。
5. 如果需要，請建立單元測試。
6. 完成前請務必通過建置 & 單元測試，並協助更新該專案資料夾根目錄下的 Readme.md 文件(該文件記錄這個專案的目的、功能說明、修改歷程等資訊)，以利後續的維運。


# 其他說明
這個專案請另外建立一個新的資料夾，並把方案檔案放在根目錄，開發的程式專案、單元測試專案另外兩的子資料夾，同時開啟開方案資料夾時可以同時看到這兩個專案，並以上述的 Git 網址進行測試。

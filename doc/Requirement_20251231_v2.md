# 需求說明
本需求針對原本的程式進行功能擴充，在原本功能中，已經可以比對不同版本 Source Code 的檔案差異，但目前通過 CI 建置後，有些 cs 檔案會被編譯成 dll 檔案，但還是有一些需要輸出的靜態檔案，例如 .js、.css、.html 等等，我還是需要手動將這些要上版的靜態檔案從`建置的目錄`複製到`預計上版的資料夾`中，但我希望可以透過自動化的方式，將這些靜態檔案複製到`預計上版的資料夾`中，詳細需求如下。

# 輸入欄位需求
1. 新增`建置結果資料夾`，需執行完比對後才允許輸入此欄位(希望可以用選擇的方式，以避免人工輸入錯誤)，此目錄包含 CI 建置完成的檔案資料夾。
2. 新增`預計上版的資料夾`，需執行完比對後才允許輸入此欄位(希望可以用選擇的方式，以避免人工輸入錯誤)，此目錄若有檔案，送出前要跳出提醒，確認是否刪除，若使用者同意才會執行複製作業。
3. 送出比對後，要將成功後的輸入欄位記錄下來，下次重新進來需記住最後輸入的資訊。
4. 輸入框要提供 placeholder 說明。


# 輸出呈現需求
1. 使用`異動清單`上面的檔案列表與`建置結果資料夾`中的檔案進行比對，若有命中，請使用相同資料夾層級複製到`預計上版的資料夾`中。
2. 完成後請跳出提醒，提醒使用者`預計上版的資料夾`中的檔案已更新完成，提醒內容包含本次複製的檔案數量 & 並提醒 dll 等二進位檔案未被複製。


# 技術需求
1. 請參考原本的程式開發風格進行調整，若要使用套件，請務必使用允許商用版本的套件。
2. 請遵循 Clean Code 的原則，另外若有特定功能的處理類別，例如處理 Git 歷程等，也請獨立類別。
3. 各類別、屬性、方法等 DOC 請使用繁體中文進行說明。
4. 如果需要，請建立單元測試。
5. 完成前請務必通過建置 & 單元測試，並協助更新該專案資料夾根目錄下的 Readme.md 文件(該文件記錄這個專案的目的、功能說明、修改歷程等資訊)，以利後續的維運。


# 其他說明
